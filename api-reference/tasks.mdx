A task represents a single execution of a workflow. Every time a workflow runs — whether triggered by a schedule, a webhook event, or a manual `/run` call — QuantumFlow creates a task record that captures the full execution history: the status at each step, any errors that occurred, timing data, and the final output produced by your agents. Use the Tasks API to poll for completion, build status dashboards, retrieve agent-generated outputs, or stream results into downstream systems.

---

## Retrieve a task

`GET /v1/tasks/{id}`

Returns the current state and full execution details of a single task.

### Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the task (e.g. `tsk_01j9xyz456`). You receive this ID when triggering a workflow via `POST /v1/workflows/{id}/run`.
</ParamField>

### Response fields

<ResponseField name="id" type="string">
  The unique system-assigned identifier for this task.
</ResponseField>

<ResponseField name="workflow_id" type="string">
  The ID of the workflow that generated this task.
</ResponseField>

<ResponseField name="status" type="string">
  The current execution status. One of:
  - `queued` — the task is waiting to be picked up by a worker.
  - `running` — the task is actively executing.
  - `completed` — all steps finished successfully.
  - `failed` — one or more steps encountered an unrecoverable error.
  - `cancelled` — the task was stopped via the cancel endpoint.
</ResponseField>

<ResponseField name="started_at" type="string">
  ISO 8601 timestamp of when execution began. `null` if the task is still `queued`.
</ResponseField>

<ResponseField name="completed_at" type="string">
  ISO 8601 timestamp of when the task reached a terminal state (`completed`, `failed`, or `cancelled`). `null` if still in progress.
</ResponseField>

<ResponseField name="duration_ms" type="integer">
  Total wall-clock execution time in milliseconds. `null` until the task completes.
</ResponseField>

<ResponseField name="steps" type="array">
  An ordered array of step result objects. Each step object contains `step_id`, `name`, `status`, `started_at`, `completed_at`, and an `output` field with the data produced by that step.
</ResponseField>

<ResponseField name="output" type="object">
  The final output of the workflow — typically the output of the last step or an aggregated result. The shape is determined by your workflow configuration.
</ResponseField>

### Request and response example

<CodeGroup>

```bash curl

  -H "Authorization: Bearer qf_live_••••••••••••••••"
```

## List tasks

`GET /v1/tasks`

Returns a paginated list of tasks across all workflows in your workspace, ordered by creation date descending. Use the filter parameters to narrow results by workflow, status, or time range.

### Query parameters

<ParamField query="workflow_id" type="string">
  Filter tasks to only those generated by the specified workflow.
</ParamField>

<ParamField query="status" type="string">
  Filter by task status. One of `queued`, `running`, `completed`, `failed`, or `cancelled`.
</ParamField>

<ParamField query="created_after" type="string">
  Return only tasks created after this ISO 8601 timestamp (e.g. `2024-01-01T00:00:00Z`).
</ParamField>

<ParamField query="created_before" type="string">
  Return only tasks created before this ISO 8601 timestamp.
</ParamField>

<ParamField query="limit" type="integer" default="20">
  Number of tasks to return per page. Maximum is `100`.
</ParamField>

<ParamField query="cursor" type="string">
  Pagination cursor from the `next_cursor` field of a previous response. Omit on the first request.
</ParamField>

### Request and response example

<CodeGroup>

```bash curl
curl "https://api.quantumflow.ai/v1/tasks?workflow_id=wf_01j9abc123&status=completed&limit=20" \
  -H "Authorization: Bearer qf_live_••••••••••••••••"
```

```python Python
import requests

response = requests.get(
    "https://api.quantumflow.ai/v1/tasks",
    headers={"Authorization": "Bearer qf_live_••••••••••••••••"},
    params={
        "workflow_id": "wf_01j9abc123",
        "status": "completed",
        "limit": 20,
    },
)
data = response.json()
tasks = data["data"]
next_cursor = data.get("next_cursor")
```

```javascript JavaScript
const params = new URLSearchParams({
  workflow_id: "wf_01j9abc123",
  status: "completed",
  limit: "20",
});
const response = await fetch(
  `https://api.quantumflow.ai/v1/tasks?${params}`,
  { headers: { "Authorization": "Bearer qf_live_••••••••••••••••" } }
);
const { data: tasks, next_cursor } = await response.json();
```

## Cancel a task

`DELETE /v1/tasks/{id}`

Cancels a task that is currently `queued` or `running`. Returns `204 No Content` on success. If the task has already reached a terminal state (`completed`, `failed`, or `cancelled`), the API returns `409 Conflict` — no changes are made.

### Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the task to cancel.
</ParamField>

<Note>
Cancelling a running task stops execution as soon as the currently active step finishes its in-flight API call. Steps that have already completed are not rolled back. Check the task's `steps` array after cancellation to see how far execution progressed before it was stopped.
</Note>

<CodeGroup>

```bash curl
curl -X DELETE https://api.quantumflow.ai/v1/tasks/tsk_01j9xyz456 \
  -H "Authorization: Bearer qf_live_••••••••••••••••"
```

```python Python
import requests

requests.delete(
    "https://api.quantumflow.ai/v1/tasks/tsk_01j9xyz456",
    headers={"Authorization": "Bearer qf_live_••••••••••••••••"},
)
# Returns 204 No Content on success
```

```javascript JavaScript
await fetch("https://api.quantumflow.ai/v1/tasks/tsk_01j9xyz456", {
  method: "DELETE",
  headers: { "Authorization": "Bearer qf_live_••••••••••••••••" },
});
// Returns 204 No Content on success
```
