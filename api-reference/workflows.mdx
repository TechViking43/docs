Workflows are the orchestration layer of QuantumFlow AI. A workflow chains together agents, integrations, and conditional logic into a repeatable automation that runs on a schedule, in response to a webhook event, or on demand. Every time a workflow executes, QuantumFlow creates a [Task](/api-reference/tasks) you can use to track progress and retrieve results. Use the Workflows API to define and version your automations in code, trigger runs from your own systems, and manage workflow lifecycle from `draft` through `active` to `paused`.

---

## Create a workflow

`POST /v1/workflows`

Creates a new workflow in `draft` status. A draft workflow can be configured and tested without running automatically. Activate it explicitly with a `PATCH` request when you are ready.

### Request body

<ParamField body="name" type="string" required>
  A descriptive name for the workflow (e.g. `"Daily Support Summary"`). Displayed in the dashboard and in Task records.
</ParamField>

<ParamField body="description" type="string">
  An optional plain-text description explaining the workflow's purpose. Useful for documentation and team collaboration.
</ParamField>

<ParamField body="trigger" type="object">
  Defines when and how the workflow starts. Contains two keys:
  - `type` (string, required): One of `schedule`, `webhook`, or `manual`.
  - `config` (object): Trigger-specific configuration. For `schedule`, provide a `cron` expression. For `webhook`, provide the expected `event` name. `manual` triggers require no config.
</ParamField>

### Request and response example

<CodeGroup>

```bash curl
curl -X POST https://api.quantumflow.ai/v1/workflows \
  -H "Authorization: Bearer qf_live_••••••••••••••••" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Daily Support Summary",
    "description": "Runs every morning at 9am on weekdays",
    "trigger": {
      "type": "schedule",
      "config": { "cron": "0 9 * * 1-5" }
    }
  }'
```

```python Python
import requests

response = requests.post(
    "https://api.quantumflow.ai/v1/workflows",
    headers={
        "Authorization": "Bearer qf_live_••••••••••••••••",
        "Content-Type": "application/json",
    },
    json={
        "name": "Daily Support Summary",
        "description": "Runs every morning at 9am on weekdays",
        "trigger": {
            "type": "schedule",
            "config": {"cron": "0 9 * * 1-5"},
        },
    },
)
workflow = response.json()
print(workflow["id"])
```

```javascript JavaScript
const response = await fetch("https://api.quantumflow.ai/v1/workflows", {
  method: "POST",
  headers: {
    "Authorization": "Bearer qf_live_••••••••••••••••",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    name: "Daily Support Summary",
    description: "Runs every morning at 9am on weekdays",
    trigger: {
      type: "schedule",
      config: { cron: "0 9 * * 1-5" },
    },
  }),
});
const workflow = await response.json();
console.log(workflow.id);
```

</CodeGroup>

```json Response 201
{
  "id": "wf_01j9abc123",
  "name": "Daily Support Summary",
  "description": "Runs every morning at 9am on weekdays",
  "status": "draft",
  "created_at": "2024-01-15T10:30:00Z"
}
```

## Trigger a workflow run

`POST /v1/workflows/{id}/run`

Manually triggers a single execution of the workflow, regardless of its configured trigger type. This is useful for testing, on-demand runs, and integrating QuantumFlow into external orchestration systems. The endpoint enqueues the run immediately and returns a `task_id` you can use to track execution via the [Tasks API](/api-reference/tasks).

### Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the workflow to run.
</ParamField>

### Request body

<ParamField body="inputs" type="object">
  An optional key-value map of runtime input variables to pass into the workflow. These are accessible to agents and step logic within the run context.
</ParamField>

### Request and response example

<CodeGroup>

```bash curl
curl -X POST https://api.quantumflow.ai/v1/workflows/wf_01j9abc123/run \
  -H "Authorization: Bearer qf_live_••••••••••••••••" \
  -H "Content-Type: application/json" \
  -d '{
    "inputs": { "date_range": "last_7_days", "team": "support" }
  }'
```

```python Python
import requests

response = requests.post(
    "https://api.quantumflow.ai/v1/workflows/wf_01j9abc123/run",
    headers={
        "Authorization": "Bearer qf_live_••••••••••••••••",
        "Content-Type": "application/json",
    },
    json={"inputs": {"date_range": "last_7_days", "team": "support"}},
)
task = response.json()
print(task["task_id"])
```

```javascript JavaScript
const response = await fetch(
  "https://api.quantumflow.ai/v1/workflows/wf_01j9abc123/run",
  {
    method: "POST",
    headers: {
      "Authorization": "Bearer qf_live_••••••••••••••••",
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      inputs: { date_range: "last_7_days", team: "support" },
    }),
  }
);
const { task_id } = await response.json();
```

</CodeGroup>

```json Response 202
{
  "task_id": "tsk_01j9xyz456",
  "workflow_id": "wf_01j9abc123",
  "status": "queued"
}
```

## Retrieve a workflow

`GET /v1/workflows/{id}`

Returns the full definition of a workflow, including its configured steps.

### Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the workflow to retrieve.
</ParamField>

### Response fields

<ResponseField name="id" type="string">
  The unique system-assigned identifier for the workflow.
</ResponseField>

<ResponseField name="name" type="string">
  The display name of the workflow.
</ResponseField>

<ResponseField name="description" type="string">
  The optional description of the workflow.
</ResponseField>

<ResponseField name="trigger" type="object">
  The trigger configuration object, including `type` and `config`.
</ResponseField>

<ResponseField name="steps" type="array">
  An ordered array of step objects that make up the workflow. Each step references an agent or integration action, along with input/output mappings.
</ResponseField>

<ResponseField name="status" type="string">
  The current lifecycle status: `draft`, `active`, or `paused`.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the workflow was created.
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp of when the workflow was last modified.
</ResponseField>

### Request and response example

<CodeGroup>

```bash curl
curl https://api.quantumflow.ai/v1/workflows/wf_01j9abc123 \
  -H "Authorization: Bearer qf_live_••••••••••••••••"
```

```python Python
import requests

response = requests.get(
    "https://api.quantumflow.ai/v1/workflows/wf_01j9abc123",
    headers={"Authorization": "Bearer qf_live_••••••••••••••••"},
)
workflow = response.json()
```

```javascript JavaScript
const response = await fetch(
  "https://api.quantumflow.ai/v1/workflows/wf_01j9abc123",
  { headers: { "Authorization": "Bearer qf_live_••••••••••••••••" } }
);
const workflow = await response.json();
```

</CodeGroup>

## Update a workflow

`PATCH /v1/workflows/{id}`

Partially updates a workflow's configuration. You can modify the `name`, `description`, `trigger`, or `status`. Only the fields you include in the request body are changed.

### Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the workflow to update.
</ParamField>

### Request body

<ParamField body="name" type="string">
  Updated display name for the workflow.
</ParamField>

<ParamField body="description" type="string">
  Updated description for the workflow.
</ParamField>

<ParamField body="trigger" type="object">
  Updated trigger configuration. Replaces the existing trigger configuration entirely.
</ParamField>

<ParamField body="status" type="string">
  Updated lifecycle status. See the status transition table below for permitted transitions.
</ParamField>

### Workflow status transitions

| From | To | Effect |
|------|----|--------|
| `draft` | `active` | Enables the configured trigger; the workflow will now run on schedule or respond to webhooks. |
| `active` | `paused` | Disables the trigger; no new runs will start until the workflow is re-activated. |
| `paused` | `active` | Re-enables the trigger. |

### Request and response example

<CodeGroup>

```bash curl
curl -X PATCH https://api.quantumflow.ai/v1/workflows/wf_01j9abc123 \
  -H "Authorization: Bearer qf_live_••••••••••••••••" \
  -H "Content-Type: application/json" \
  -d '{ "status": "active" }'
```

```python Python
import requests

response = requests.patch(
    "https://api.quantumflow.ai/v1/workflows/wf_01j9abc123",
    headers={
        "Authorization": "Bearer qf_live_••••••••••••••••",
        "Content-Type": "application/json",
    },
    json={"status": "active"},
)
workflow = response.json()
```

```javascript JavaScript
const response = await fetch(
  "https://api.quantumflow.ai/v1/workflows/wf_01j9abc123",
  {
    method: "PATCH",
    headers: {
      "Authorization": "Bearer qf_live_••••••••••••••••",
      "Content-Type": "application/json",
    },
    body: JSON.stringify({ status: "active" }),
  }
);
const workflow = await response.json();
```

</CodeGroup>

```json Response 200
{
  "id": "wf_01j9abc123",
  "name": "Daily Support Summary",
  "status": "active",
  "updated_at": "2024-01-16T09:00:00Z"
}
```

## Delete a workflow

`DELETE /v1/workflows/{id}`

Permanently deletes a workflow and all of its step configuration. Returns `204 No Content` on success.

### Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the workflow to delete.
</ParamField>

<Warning>
You must pause an active workflow before deleting it. Attempting to delete a workflow with `status: active` returns a `409 Conflict` error. Pause the workflow first with a `PATCH` request, then delete it. Note that deleting a workflow does not delete its historical Task records — those remain queryable via the Tasks API.
</Warning>

<CodeGroup>

```bash curl
curl -X DELETE https://api.quantumflow.ai/v1/workflows/wf_01j9abc123 \
  -H "Authorization: Bearer qf_live_••••••••••••••••"
```

```python Python
import requests

requests.delete(
    "https://api.quantumflow.ai/v1/workflows/wf_01j9abc123",
    headers={"Authorization": "Bearer qf_live_••••••••••••••••"},
)
# Returns 204 No Content on success
```

```javascript JavaScript
await fetch("https://api.quantumflow.ai/v1/workflows/wf_01j9abc123", {
  method: "DELETE",
  headers: { "Authorization": "Bearer qf_live_••••••••••••••••" },
});
// Returns 204 No Content on success
```

</CodeGroup>

## List workflows

`GET /v1/workflows`

Returns a paginated list of all workflows in your workspace, ordered by creation date descending.

### Query parameters

<ParamField query="limit" type="integer" default="20">
  Number of workflows to return per page. Maximum is `100`.
</ParamField>

<ParamField query="cursor" type="string">
  Pagination cursor from the `next_cursor` field of the previous response. Omit on the first request.
</ParamField>

### Request and response example

<CodeGroup>

```bash curl
curl "https://api.quantumflow.ai/v1/workflows?limit=20" \
  -H "Authorization: Bearer qf_live_••••••••••••••••"
```

```python Python
import requests

response = requests.get(
    "https://api.quantumflow.ai/v1/workflows",
    headers={"Authorization": "Bearer qf_live_••••••••••••••••"},
    params={"limit": 20},
)
data = response.json()
workflows = data["data"]
next_cursor = data.get("next_cursor")
```

```javascript JavaScript
const response = await fetch(
  "https://api.quantumflow.ai/v1/workflows?limit=20",
  { headers: { "Authorization": "Bearer qf_live_••••••••••••••••" } }
);
const { data: workflows, next_cursor } = await response.json();
```

</CodeGroup>

```json Response 200
{
  "data": [
    {
      "id": "wf_01j9abc123",
      "name": "Daily Support Summary",
      "status": "active",
      "created_at": "2024-01-15T10:30:00Z"
    }
  ],
  "next_cursor": null
}
```
