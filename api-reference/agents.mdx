Agents are the core intelligence units in QuantumFlow AI. Each agent is a configured AI model paired with a set of instructions, an optional toolset, and optional escalation rules. You deploy agents inside workflow steps where they reason over data, call integrations, and produce structured outputs. Use the Agents API to create and iterate on agents programmatically, wire them into CI/CD pipelines, or manage large fleets of specialized agents at scale.

---

## Create an agent

`POST /v1/agents`

Creates a new agent and returns it with a system-assigned `id`. The agent starts in `active` status and is immediately available to be referenced in workflow steps.

### Request body

<ParamField body="name" type="string" required>
  A human-readable display name for the agent (e.g. `"Support Ticket Analyzer"`). Used in the dashboard and in workflow step labels.
</ParamField>

<ParamField body="instructions" type="string" required>
  The system prompt that defines the agent's role, goal, and output format. Be specific — the quality of the instructions directly determines the quality of the agent's output.
</ParamField>

<ParamField body="model" type="string" default="gpt-4o">
  The underlying AI model to use. Accepted values: `gpt-4o`, `gpt-4o-mini`, `claude-3-5-sonnet`. Defaults to `gpt-4o` if omitted.
</ParamField>

<ParamField body="tools" type="array">
  An array of integration IDs (e.g. `["int_zendesk_01", "int_slack_01"]`) representing the third-party tools this agent is permitted to call during execution. Integration IDs are returned by the [Integrations API](/api-reference/integrations).
</ParamField>

<ParamField body="escalation" type="object">
  Optional escalation configuration. When `enabled` is `true`, the agent routes outputs that require human review to the specified channel. Example: `{ "enabled": true, "channel": "slack_channel_id" }`.
</ParamField>

### Request and response example

<CodeGroup>

```bash curl
curl -X POST https://api.quantumflow.ai/v1/agents \
  -H "Authorization: Bearer qf_live_••••••••••••••••" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Support Ticket Analyzer",
    "instructions": "You analyze support tickets and summarize trends. Output a bullet list of top issues.",
    "model": "gpt-4o",
    "tools": ["int_zendesk_01", "int_slack_01"]
  }'
```

```python Python
import requests

response = requests.post(
    "https://api.quantumflow.ai/v1/agents",
    headers={
        "Authorization": "Bearer qf_live_••••••••••••••••",
        "Content-Type": "application/json",
    },
    json={
        "name": "Support Ticket Analyzer",
        "instructions": "You analyze support tickets and summarize trends. Output a bullet list of top issues.",
        "model": "gpt-4o",
        "tools": ["int_zendesk_01", "int_slack_01"],
    },
)
agent = response.json()
print(agent["id"])
```

```javascript JavaScript
const response = await fetch("https://api.quantumflow.ai/v1/agents", {
  method: "POST",
  headers: {
    "Authorization": "Bearer qf_live_••••••••••••••••",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    name: "Support Ticket Analyzer",
    instructions: "You analyze support tickets and summarize trends. Output a bullet list of top issues.",
    model: "gpt-4o",
    tools: ["int_zendesk_01", "int_slack_01"],
  }),
});
const agent = await response.json();
console.log(agent.id);
```

</CodeGroup>

```json Response 201
{
  "id": "agt_01j9def456",
  "name": "Support Ticket Analyzer",
  "model": "gpt-4o",
  "status": "active",
  "created_at": "2024-01-15T10:30:00Z"
}
```

## Retrieve an agent

`GET /v1/agents/{id}`

Returns the full details of a single agent identified by its `id`.

### Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the agent (e.g. `agt_01j9def456`).
</ParamField>

### Response fields

<ResponseField name="id" type="string">
  The unique system-assigned identifier for the agent.
</ResponseField>

<ResponseField name="name" type="string">
  The display name of the agent.
</ResponseField>

<ResponseField name="instructions" type="string">
  The system prompt / goal configured for the agent.
</ResponseField>

<ResponseField name="model" type="string">
  The AI model assigned to the agent (e.g. `gpt-4o`).
</ResponseField>

<ResponseField name="tools" type="array">
  List of integration IDs the agent is permitted to use.
</ResponseField>

<ResponseField name="escalation" type="object">
  The escalation configuration object, or `null` if escalation is not configured.
</ResponseField>

<ResponseField name="status" type="string">
  Current status of the agent: `active` or `archived`.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the agent was created.
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp of when the agent was last updated.
</ResponseField>

### Request and response example

<CodeGroup>

```bash curl
curl https://api.quantumflow.ai/v1/agents/agt_01j9def456 \
  -H "Authorization: Bearer qf_live_••••••••••••••••"
```

```python Python
import requests

response = requests.get(
    "https://api.quantumflow.ai/v1/agents/agt_01j9def456",
    headers={"Authorization": "Bearer qf_live_••••••••••••••••"},
)
agent = response.json()
```

```javascript JavaScript
const response = await fetch(
  "https://api.quantumflow.ai/v1/agents/agt_01j9def456",
  { headers: { "Authorization": "Bearer qf_live_••••••••••••••••" } }
);
const agent = await response.json();
```

</CodeGroup>

```json Response 200
{
  "id": "agt_01j9def456",
  "name": "Support Ticket Analyzer",
  "instructions": "You analyze support tickets and summarize trends. Output a bullet list of top issues.",
  "model": "gpt-4o",
  "tools": ["int_zendesk_01", "int_slack_01"],
  "escalation": null,
  "status": "active",
  "created_at": "2024-01-15T10:30:00Z",
  "updated_at": "2024-01-15T10:30:00Z"
}
```

## Update an agent

`PATCH /v1/agents/{id}`

Partially updates an existing agent. Include only the fields you want to change — all other fields remain unchanged. You can update any field that is accepted by the create endpoint.

### Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the agent to update.
</ParamField>

### Request body

<ParamField body="name" type="string">
  Updated display name for the agent.
</ParamField>

<ParamField body="instructions" type="string">
  Updated system prompt for the agent.
</ParamField>

<ParamField body="model" type="string">
  Updated AI model. Accepted values: `gpt-4o`, `gpt-4o-mini`, `claude-3-5-sonnet`.
</ParamField>

<ParamField body="tools" type="array">
  Updated list of integration IDs the agent is permitted to use. Replaces the existing list entirely.
</ParamField>

<ParamField body="escalation" type="object">
  Updated escalation configuration. Set `enabled` to `false` to disable escalation.
</ParamField>

### Request and response example

<CodeGroup>

```bash curl
curl -X PATCH https://api.quantumflow.ai/v1/agents/agt_01j9def456 \
  -H "Authorization: Bearer qf_live_••••••••••••••••" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "claude-3-5-sonnet",
    "escalation": { "enabled": true, "channel": "C04ABC123" }
  }'
```

```python Python
import requests

response = requests.patch(
    "https://api.quantumflow.ai/v1/agents/agt_01j9def456",
    headers={
        "Authorization": "Bearer qf_live_••••••••••••••••",
        "Content-Type": "application/json",
    },
    json={
        "model": "claude-3-5-sonnet",
        "escalation": {"enabled": True, "channel": "C04ABC123"},
    },
)
agent = response.json()
```

```javascript JavaScript
const response = await fetch(
  "https://api.quantumflow.ai/v1/agents/agt_01j9def456",
  {
    method: "PATCH",
    headers: {
      "Authorization": "Bearer qf_live_••••••••••••••••",
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      model: "claude-3-5-sonnet",
      escalation: { enabled: true, channel: "C04ABC123" },
    }),
  }
);
const agent = await response.json();
```

</CodeGroup>

```json Response 200
{
  "id": "agt_01j9def456",
  "name": "Support Ticket Analyzer",
  "model": "claude-3-5-sonnet",
  "status": "active",

}
```

## Delete an agent

`DELETE /v1/agents/{id}`

Permanently deletes an agent. Returns `204 No Content` on success. This action cannot be undone.

### Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the agent to delete.
</ParamField>

<Warning>
Deleting an agent that is referenced by one or more active workflow steps will cause those steps to fail at runtime. Before deleting an agent, pause or update any workflows that depend on it, or replace the agent reference in those steps with an alternative agent.
</Warning>

<CodeGroup>

```bash curl
curl -X DELETE https://api.quantumflow.ai/v1/agents/agt_01j9def456 \
  -H "Authorization: Bearer qf_live_••••••••••••••••"
```

```python Python
import requests

requests.delete(
    "https://api.quantumflow.ai/v1/agents/agt_01j9def456",
    headers={"Authorization": "Bearer qf_live_••••••••••••••••"},
)
# Returns 204 No Content on success
```

```javascript JavaScript
await fetch("https://api.quantumflow.ai/v1/agents/agt_01j9def456", {
  method: "DELETE",
  headers: { "Authorization": "Bearer qf_live_••••••••••••••••" },
});
// Returns 204 No Content on success
```

</CodeGroup>

## List agents

`GET /v1/agents`

Returns a paginated list of all agents in your workspace, ordered by creation date descending.

### Query parameters

<ParamField query="limit" type="integer" default="20">
  Number of agents to return per page. Maximum is `100`.
</ParamField>

<ParamField query="cursor" type="string">
  Opaque pagination cursor returned as `next_cursor` in the previous response. Omit this parameter on the first request.
</ParamField>

### Request and response example

<CodeGroup>

```bash curl
curl "https://api.quantumflow.ai/v1/agents?limit=20" \
  -H "Authorization: Bearer qf_live_••••••••••••••••"
```

```python Python
import requests

response = requests.get(
    "https://api.quantumflow.ai/v1/agents",
    headers={"Authorization": "Bearer qf_live_••••••••••••••••"},
    params={"limit": 20},
)
data = response.json()
agents = data["data"]
next_cursor = data.get("next_cursor")
```

```javascript JavaScript
const response = await fetch(
  "https://api.quantumflow.ai/v1/agents?limit=20",
  { headers: { "Authorization": "Bearer qf_live_••••••••••••••••" } }
);
const { data: agents, next_cursor } = await response.json();
```

</CodeGroup>

```json Response 200
{
  "data": [
    {
      "id": "agt_01j9def456",
      "name": "Support Ticket Analyzer",
      "model": "gpt-4o",
      "status": "active",
      "created_at": "2024-01-15T10:30:00Z"
    }
  ],
  "next_cursor": "cur_01j9def999"
}
```
