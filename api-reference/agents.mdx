Agents are the core intelligence units in QuantumFlow AI. Each agent is a configured AI model paired with a set of instructions, an optional toolset, and optional escalation rules. You deploy agents inside workflow steps where they reason over data, call integrations, and produce structured outputs. Use the Agents API to create and iterate on agents programmatically, wire them into CI/CD pipelines, or manage large fleets of specialized agents at scale.

---

## Create an agent

<api-endpoint method="POST" path="/v1/agents" />

Creates a new agent and returns it with a system-assigned `id`. The agent starts in `active` status and is immediately available to be referenced in workflow steps.

### Request body

<ParamField body="name" type="string" required>
  A human-readable display name for the agent (e.g. `"Support Ticket Analyzer"`). Used in the dashboard and in workflow step labels.
</ParamField>

<ParamField body="instructions" type="string" required>
  The system prompt that defines the agent's role, goal, and output format. Be specific — the quality of the instructions directly determines the quality of the agent's output.
</ParamField>

<ParamField body="model" type="string" default="gpt-4o">
  The underlying AI model to use. Accepted values: `gpt-4o`, `gpt-4o-mini`, `claude-3-5-sonnet`. Defaults to `gpt-4o` if omitted.
</ParamField>

<ParamField body="tools" type="array">
  An array of integration IDs (e.g. `["int_zendesk_01", "int_slack_01"]`) representing the third-party tools this agent is permitted to call during execution. Integration IDs are returned by the [Integrations API](/api-reference/integrations).
</ParamField>

<ParamField body="escalation" type="object">
  Optional escalation configuration. When `enabled` is `true`, the agent routes outputs that require human review to the specified channel.

  ```json
  { "enabled": true, "channel": "slack_channel_id" }
  ```
</ParamField>

### Request and response example

<CodeGroup>

```json Request
{
  "name": "Support Ticket Analyzer",
  "instructions": "You analyze support tickets and summarize trends. Output a bullet list of top issues.",
  "model": "gpt-4o",
  "tools": ["int_zendesk_01", "int_slack_01"]
}
```

```json Response 201
{
  "id": "agt_01j9def456",
  "name": "Support Ticket Analyzer",
  "model": "gpt-4o",
  "status": "active",
  "created_at": "2024-01-15T10:30:00Z"
}
```

</CodeGroup>

## Retrieve an agent

<api-endpoint method="GET" path="/v1/agents/{id}" />

Returns the full details of a single agent identified by its `id`.

### Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the agent (e.g. `agt_01j9def456`).
</ParamField>

### Response fields

<ResponseField name="id" type="string">
  The unique system-assigned identifier for the agent.
</ResponseField>

<ResponseField name="name" type="string">
  The display name of the agent.
</ResponseField>

<ResponseField name="instructions" type="string">
  The system prompt / goal configured for the agent.
</ResponseField>

<ResponseField name="model" type="string">
  The AI model assigned to the agent (e.g. `gpt-4o`).
</ResponseField>

<ResponseField name="tools" type="array">
  List of integration IDs the agent is permitted to use.
</ResponseField>

<ResponseField name="escalation" type="object">
  The escalation configuration object, or `null` if escalation is not configured.
</ResponseField>

<ResponseField name="status" type="string">
  Current status of the agent: `active` or `archived`.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the agent was created.
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp of when the agent was last updated.
</ResponseField>

## Update an agent

<api-endpoint method="PATCH" path="/v1/agents/{id}" />

Partially updates an existing agent. Include only the fields you want to change — all other fields remain unchanged. You can update any field that is accepted by the create endpoint.

### Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the agent to update.
</ParamField>

### Request and response example

<CodeGroup>

```json Request
{
  "model": "claude-3-5-sonnet",
  "escalation": { "enabled": true, "channel": "C04ABC123" }
}
```

```json Response 200
{
  "id": "agt_01j9def456",
  "name": "Support Ticket Analyzer",
  "model": "claude-3-5-sonnet",
  "status": "active",
  "updated_at": "2024-01-16T08:15:00Z"
}
```

</CodeGroup>

## Delete an agent

<api-endpoint method="DELETE" path="/v1/agents/{id}" />

Permanently deletes an agent. Returns `204 No Content` on success. This action cannot be undone.

### Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the agent to delete.
</ParamField>

<Warning>
Deleting an agent that is referenced by one or more active workflow steps will cause those steps to fail at runtime. Before deleting an agent, pause or update any workflows that depend on it, or replace the agent reference in those steps with an alternative agent.
</Warning>

## List agents

<api-endpoint method="GET" path="/v1/agents" />

Returns a paginated list of all agents in your workspace, ordered by creation date descending.

### Query parameters

<ParamField query="limit" type="integer" default="20">
  Number of agents to return per page. Maximum is `100`.
</ParamField>

<ParamField query="cursor" type="string">
  Opaque pagination cursor returned as `next_cursor` in the previous response. Omit this parameter on the first request.
</ParamField>

### Response example

```json
{
  "data": [
    {
      "id": "agt_01j9def456",
      "name": "Support Ticket Analyzer",
      "model": "gpt-4o",
      "status": "active",
      "created_at": "2024-01-15T10:30:00Z"
    }
  ],
  "next_cursor": "cur_01j9def999"
}
```
